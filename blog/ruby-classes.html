Who is a Class & What Does it do?(Arnold Voice)

FJ Collins Jr. - February 7, 2016

So you have begun your own journey through Ruby and understand how almost everything in Ruby is an object - arrays, hashes, integers, strings etc. Each one of these elements have a separate class to associate them with. For example, an easy way to find out which object belongs to which class is to call class on that object... 


			puts 1.class
			puts "".class
			puts [].class

			=>Fixnum
			=>String
			=>Array
You can also ask whether a specific object is part of a particular class... 

			puts 1.is_a?(Integer)
			puts 1.is_a?(Array)

			=> true
			=>false
This is all well and good, but how can we manipulate classes to see how a real-life object works? What if we have an object like computer and we want to see its attributes, responsibilities, actions etc. We can create our own class called Computer and create different methods of objects to see its actions, color, size etc.

Creating a Class for Computer

Like methods, classes are defined with opening word class and is completed with an end statement. 


	    	class Computer
	    	end
Next, we want to indicate which variables are going to be accessed throughout the class. A best-practice use would be to Initialize the variables for computers. 

	    	class Computer
	    		def Initialize(brand, model)
	    			@brand = brand
	    			@model = model_number
	    			@os = operating_system
	    			@users = users
	    		end
	    	end
Here we have a couple of variables defined and you might have noticed the @ before each variable. These are known as instance variables and their scope are confined to whatever object self refers to. Therefore, although it is a variable, @os can be referenced throughout the class Computer.

This concept of instantiation sounds intimidating, however it's really a not as bas as it seems. This just means that when Ruby uses the Computer class to create a new one, that computer should come pre-loaded with certain information in the form of instance variables.

Next we create some methods of the computer - we might want to see which user is currently on the computer, what documents are opened, what programs are being used etc... 

	    	class Computer
	    		def Initialize(brand, model, os, user)
	    			@brand = brand
	    			@model = model_number
	    			@os = operating_system
	    			@user = user
	    		end

	    		def current_user
	    			@user
	    		end
	    	end
As simple as #current_user is, it will display which user is currently using the computer. Now a pretty neat trick to use instead of just displaying the current user is to use the attr_reader, attr_writer and attr_accessor tools.
  • attr_reader is a reader which means that it just returns a value or state outside of the class, but does not change it.
  • attr_writer is a writer which means that you can change the value of the variable outside of the class, but it is not readable.
  • attr_accessor is both a reader and a writer. It can be used to return and change the value of the variable.
Therefore, our new code would look like this... 

	    	class Computer
	    		attr_reader :user
	    		def Initialize(brand, model, os, user)
	    			@brand = brand
	    			@model = model_number
	    			@os = operating_system
	    			@user = user
	    		end

	    		
	    	end

I hope this gives you a general understand of Classes and how they work with attributes and methods. Please come back soon!
