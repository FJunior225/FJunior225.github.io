How to Secure Your Code

Journey Through DBC

FJ Collins Jr. - January 24th, 2016

Being new to not only the Tech Industry, but to the Programming Industry as well, there are a lot of things I do not know. One for which, is security. More specifically how developers follow best practices when it comes to securing their programs.

Where does insecurities come from?

Plain and simple, bugs. Why do programs have bugs? We're all human and make mistakes, whether it be from ignorance, haste, carelessness or any other reason. No one is perfect, therefore, no code is necessarily perfect. These bugs can lead to unintentional flaws that allow someone to compromise your system and take advantage of those human errors. So what can we do to prevent this from happening?

Best-Practices for coding

  1. Design your program before you code: It's better to have some sort of structure before you begin your code, so that changes made will affect little code. More changes could lead to more mistakes made, nonetheless, you should think about such things like the environment in which the program will run; input and output behavior; any files to be used; arguments recognized; or any other aspects of behavior. Realizing these behaviors can recognize future errors that may arise and a way to fix them.
  2. Make critical portion of your program as simple as possible: This will help to ensure that any debugging will not hinder your program functions. Simple coding will also lead to less errors and in turn, less bugs.
  3. Never trust just any input: This is important because untrusted sources can lead to damaging code. Make sure you're feedback is from people you trust or websites you trust.
  4. Resist the temptation to add a feature for the sole purpose that you can: This stems from keep your code simple - only add a feature when it is identified that it is needed.
  5. Lastly, keep it simple again!

I note only wrote this post for others to gain knowledge of best-practices when it comes to securely code, but also for myself. It was an interesting topic to research and it reminded me that simple code is secure code. DBC is really doing a good job having us research concepts we enjoy and don't understand fully. I'm excited to see what comes up next!
